# PaiCLI 使用说明文档

## 一、项目概述

PaiCLI 是一个成熟的 Java Agent CLI 产品，从第一期的 ReAct 单代理循环逐步演进到完整的智能对话系统。项目的核心亮点是**拟人化对话引擎**——与用户对话的 AI 角色具有自己的"生命"，会根据日程安排主动与用户交流。

### 主要特性

- 🤖 **多模式对话**：ReAct、Plan-and-Execute、Multi-Agent 协作
- 🧠 **记忆系统**：短期记忆、长期记忆、上下文压缩
- 🔍 **代码理解**：RAG 语义检索、代码关系图谱
- 🌐 **联网能力**：web_search、web_fetch、浏览器 MCP
- ⏰ **定时任务**：每日循环任务、一次性提醒
- 💬 **主动对话**：拟人化心跳扫描、智能触发
- 📅 **人生计划**：五级人生规划管理（Life/Year/Month/Week/Day）
- 🎭 **角色系统**：多角色切换、小说角色提取

## 二、快速开始

### 2.1 环境要求

- Java 17+
- Maven 3.8+
- API Key（GLM、DeepSeek、StepFun、Kimi 或 FreeLLMAPI 任选其一）

### 2.2 配置 API Key

复制 `.env.example` 为 `.env`，填入你的 API Key：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 示例：使用 StepFun
STEP_API_KEY=your_step_api_key_here
STEP_MODEL=step-3.5-flash

# 或使用 GLM
GLM_API_KEY=your_glm_api_key_here

# 或使用 Kimi
KIMI_API_KEY=your_kimi_api_key_here
KIMI_MODEL=kimi-k2.6
```

### 2.3 编译运行

```bash
# 编译（默认跳过测试）
mvn clean package

# 运行
java -jar target/paicli-1.0-SNAPSHOT.jar
```

或直接运行：

```bash
mvn clean compile exec:java -Dexec.mainClass="com.paicli.cli.Main"
```

## 三、核心功能详解

### 3.1 角色系统

#### 3.1.1 创建角色

PaiCLI 支持三种方式创建角色：

**方式一：从小说提取配角性格（仅性格，不经历故事）**

```text
/创建角色 林黛玉 @红楼梦.txt
```

此方式使用 `persona-extraction` Skill，只提取角色的性格特征，不包含小说中的故事动线。生成的角色会有一套与小说无关的虚构人生计划。

**方式二：从小说提取主角生命历程（体验完整故事）**

```text
/创建角色 福贵 @活着.txt
```

此方式使用 `character-extraction` Skill，提取角色的完整生命历程，包括阶段划分、关键事件、性格变化等。角色会经历小说中的故事。

**方式三：完全自定义人格**

直接创建角色目录并手动编写角色文件，或通过对话让 AI 帮你生成：

```text
/创建角色 我的专属AI
```

#### 3.1.2 角色文件结构

每个角色的数据存储在 `.paicli/souls/{角色名}/` 目录下：

| 文件 | 内容 |
|------|------|
| `soul.md` | 角色设定（性格、说话风格、行为模式等） |
| `timeline-life.md` | 生命历程（仅主角提取模式有） |
| `life-plan.md` | 人生规划（5-10年大局） |
| `year-plan.md` | 年度计划 |
| `month-plan.md` | 月度计划 |
| `week-plan.md` | 周计划 |
| `day-plan.md` | 今日计划 |
| `logs/YYYY-MM-DD.md` | 每日日志 |

#### 3.1.3 角色管理命令

```text
/角色列表              # 查看所有角色
/切换角色 福贵         # 切换到指定角色
/人格                  # 查看当前角色信息和人生计划
```

### 3.2 人生计划系统

人生计划采用五级层级管理：

```
Life Plan（人生规划）
    └── Year Plan（年度计划）
            └── Month Plan（月度计划）
                    └── Week Plan（周计划）
                            └── Day Plan（今日计划）
```

#### 3.2.1 查看人生计划

```text
/人生计划              # 查看完整的人生计划
```

输出示例：

```text
【人生规划】
我要成为一个自由自在的人，走遍天涯海角，体验各种生活。

【今年计划】
1. 上半年：学习钓鱼技巧
2. 下半年：去海边生活一段时间

【本月计划】
- 每周去河边练习钓鱼
- 准备海边旅行的行李

【本周计划】
周一到周五：上午钓鱼，下午休息
周末：整理渔具

【今日安排】
- 上午：去河边钓鱼
- 中午：在河边吃午饭
- 下午：回家休息
```

#### 3.2.2 计划更新机制

系统每天凌晨 3 点会自动执行 `daily-worker` 任务，根据角色当前状态更新计划：

- **每日**：生成新的 `day-plan.md`，编写今日日志
- **每周一**：更新 `week-plan.md`
- **每月1号**：更新 `month-plan.md`
- **每年1月1号**：更新 `year-plan.md`

### 3.3 定时任务系统

#### 3.3.1 任务类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `daily` | 每日循环任务 | 每天早上 8 点提醒用户 |
| `once` | 一次性任务 | 2026年6月20日下午3点提醒 |

#### 3.3.2 任务管理命令

```text
/定时任务              # 查看所有定时任务
/任务运行 daily-worker # 立即执行指定任务
```

#### 3.3.3 系统任务

系统默认会创建以下定时任务：

| 任务ID | 时间 | 功能 |
|--------|------|------|
| `daily-worker` | 每天 03:00 | 日常模拟，更新计划和日志 |

#### 3.3.4 任务存储

任务以 `.task` 文件存储在 `.paicli/tasks/` 目录下：

```
id=daily-worker
type=daily
hour=3
minute=0
next_run=2026-06-18T03:00
system=true
---
你是 Worker。先调 load_skill("daily-simulation") 加载日常模拟 Skill...
```

### 3.4 主动对话系统

主动对话是 PaiCLI 的核心特色，让 AI 角色像真人一样主动开启话题。

#### 3.4.1 触发规则

| 触发类型 | 条件 | 说明 |
|----------|------|------|
| **时段触发** | 早8-9点、午12-13点、晚20-22点 | 每个时段最多1次 |
| **空闲触发** | 用户超过30分钟无对话 | 每天最多2次 |
| **被动失效** | 用户主动对话后 | 每轮最多1次 |

#### 3.4.2 频率控制

- **每日上限**：每天最多主动3次
- **冷却期**：主动后5轮对话内不再次主动
- **连续忽略**：连续2次被忽略，当天频率减半
- **禁止时段**：凌晨0:00-7:00不触发

#### 3.4.3 主动对话命令

```text
/主动状态              # 查看主动对话状态
/静音 1                # 静音1小时
/常态                  # 恢复默认主动触发
```

状态输出示例：

```text
主动对话状态：🔔开启 | 今日主动：1/3 | 连续忽略：0 | 冷却轮数：0 | 最后用户消息：14:30
```

#### 3.4.4 主动对话示例

```text
💬 今天天气不错，我早上出去钓鱼了，收获还不错呢！
```

## 四、使用流程

### 4.1 完整使用流程

```
1. 启动 PaiCLI
       ↓
2. 创建角色
   ├── /创建角色 福贵 @活着.txt      （体验生命历程）
   ├── /创建角色 林黛玉 @红楼梦.txt  （仅提取性格）
   └── /创建角色 我的AI              （自定义）
       ↓
3. 查看角色信息
   ├── /人格                        （查看角色设定和计划）
   └── /人生计划                    （查看完整计划）
       ↓
4. 日常对话
   ├── 普通聊天（ReAct模式）
   ├── /plan 复杂任务（计划模式）
   └── /team 协作任务（多Agent模式）
       ↓
5. 体验主动对话
   ├── 等待角色主动开启话题
   ├── /主动状态 查看状态
   └── /静音 暂时关闭主动对话
       ↓
6. 查看日志
   └── 角色会自动记录每日生活日志
```

### 4.2 角色初始化流程

当创建新角色时，系统会执行以下步骤：

#### 方式一：主角生命历程（character-extraction）

```
上传小说 → 分块扫描 → 提取动线和性格 → 生成 timeline-life.md 和 soul.md → 
初始化五级计划文件 → 开始日常模拟
```

#### 方式二：配角性格提取（persona-extraction）

```
上传小说 → 识别目标角色 → 仅提取性格特征 → 生成 soul.md（不含故事）→ 
生成虚构人生计划 → 开始日常模拟
```

#### 方式三：完全自定义

```
创建角色目录 → 编写 soul.md → 生成人生计划 → 开始日常模拟
```

### 4.3 日常模拟流程

每天凌晨 3 点，`daily-worker` 任务会自动执行：

```
1. 检查计划文件是否需要更新
2. 读取 soul.md 和 day-plan.md
3. 根据角色性格和今日计划生成日常活动
4. 编写今日日志（logs/YYYY-MM-DD.md）
5. 更新明日计划
6. 如有重大事件，创建一次性定时任务通知用户
```

## 五、技术实现

### 5.1 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                       PaiCLI 主程序                          │
├─────────────────────────────────────────────────────────────┤
│  CLI 层（Main.java）                                         │
│    ├── 命令解析（CliCommandParser）                          │
│    ├── 终端交互（JLine）                                     │
│    └── 渲染器（Renderer：inline/lanterna/plain）             │
├─────────────────────────────────────────────────────────────┤
│  Agent 层                                                   │
│    ├── ReAct Agent                                          │
│    ├── Plan-and-Execute Agent                               │
│    └── Multi-Agent（Planner/Worker/Reviewer）               │
├─────────────────────────────────────────────────────────────┤
│  Runtime 层（后台服务）                                       │
│    ├── ScheduledTaskManager    ← 定时任务调度（每15秒tick）   │
│    ├── ProactiveScheduler      ← 主动对话调度（每60秒tick）   │
│    └── LifePlanManager         ← 人生计划管理                │
├─────────────────────────────────────────────────────────────┤
│  数据存储                                                    │
│    ├── .paicli/souls/{角色名}/  ← 角色数据                   │
│    ├── .paicli/tasks/           ← 定时任务                   │
│    └── ~/.paicli/               ← 用户级配置和记忆            │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 定时任务实现

**文件：** `src/main/java/com/paicli/runtime/task/ScheduledTaskManager.java`

**核心机制：**

1. 使用 `ScheduledExecutorService` 创建守护线程，每 15 秒执行一次 `tick()` 方法
2. `tick()` 方法扫描 `.paicli/tasks/` 目录下所有 `.task` 文件
3. 检查每个任务的 `next_run` 是否小于等于当前时间
4. 触发到期任务，执行后更新下次运行时间
5. 一次性任务执行后自动删除

**关键代码：**

```java
// 每15秒检查一次
scheduler.scheduleAtFixedRate(this::tick, 0, 15, TimeUnit.SECONDS);

private void tick() {
    LocalDateTime now = LocalDateTime.now();
    // 扫描所有任务文件，检查是否到期
    // 执行到期任务...
}
```

### 5.3 主动对话实现

**文件：** `src/main/java/com/paicli/runtime/proactive/ProactiveScheduler.java`

**核心机制：**

1. 使用 `ScheduledExecutorService` 创建守护线程，每 60 秒执行一次 `tick()` 方法
2. `tick()` 方法依次检查：
   - 是否在禁止时段（凌晨0:00-7:00）
   - 是否静音
   - 是否在冷却期
   - 是否超过每日上限
   - 是否满足触发条件（时段触发或空闲触发）
3. 满足条件时，调用 LLM 生成主动消息
4. 通过监听器机制推送到 CLI

**关键代码：**

```java
// 每60秒检查一次
scheduler.scheduleAtFixedRate(this::tick, 0, TICK_INTERVAL_SECONDS, TimeUnit.SECONDS);

private void tick() {
    // 检查静音、时段、冷却、日上限...
    TriggerType trigger = checkTrigger(now, hour);
    if (trigger != null) {
        generateAndSendProactiveMessage(trigger);
    }
}
```

### 5.4 人生计划实现

**文件：** `src/main/java/com/paicli/runtime/life/LifePlanManager.java`

**核心机制：**

1. 管理角色的五级人生计划文件
2. 支持多角色切换和管理
3. 提供 `getLifeContext()` 方法，将五级计划整合为一个完整的上下文字符串
4. 上下文会注入到 LLM 的 system prompt 中，让角色了解自己的"人生"

**关键代码：**

```java
public String getLifeContext() {
    StringBuilder sb = new StringBuilder();
    sb.append("【人生规划】\n").append(getLifePlan()).append("\n\n");
    sb.append("【今年计划】\n").append(getYearPlan()).append("\n\n");
    sb.append("【本月计划】\n").append(getMonthPlan()).append("\n\n");
    sb.append("【本周计划】\n").append(getWeekPlan()).append("\n\n");
    sb.append("【今日安排】\n").append(getDayPlan()).append("\n");
    return sb.toString();
}
```

## 八、常见问题

### Q1：主动对话没有触发？

A：请检查：
1. 是否在允许时段（8:00-22:00）
2. 是否超过每日上限（/主动状态 查看）
3. 是否在静音状态（/主动状态 查看）
4. 是否刚主动过还在冷却期

### Q2：人生计划没有显示？

A：请检查：
1. 是否创建了角色（/角色列表）
2. 角色目录下是否有计划文件（`.paicli/souls/{角色名}/`）
3. 运行 `/人格` 命令查看角色信息

### Q3：如何创建完全自定义的角色？

A：有两种方式：
1. 手动创建目录 `.paicli/souls/你的角色名/`，然后编写 `soul.md` 和各计划文件
2. 使用 `/创建角色 你的角色名`，然后通过对话让 AI 帮你完善

### Q4：定时任务不执行？

A：请检查：
1. `/定时任务` 查看任务是否存在
2. 任务的 `next_run` 时间是否已过
3. 使用 `/任务运行 <任务ID>` 手动触发测试

### Q5：如何修改角色的性格？

A：进入管理模式后修改 `soul.md` 文件：
```text
/管理
# 然后使用 write_file 工具修改 .paicli/souls/{角色名}/soul.md
/管理  # 退出管理模式
```
