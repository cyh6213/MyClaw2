# 定时任务-Skill

## 两部分：创建规则 + 执行规则

---

## 第一部分：创建定时任务（AI 对用户）

当用户说"提醒我XXX"、"设个提醒"之类的话时适用。

### 严格原则
- 用户要求设几个就只设几个，不多设
- 设完就结束，不要继续查看任务列表、不要额外添加每日任务
- 如果用户说"五分钟后提醒喝水"，就只创建一个 type=once 的一次性任务
- 如果用户说"每天早上8点提醒喝水"，就创建一个 type=daily 的每日任务
- 不要替用户做决定——用户没说要每日提醒，就不要自动创建每日循环

### 时间解析
- `type=daily`：time 填 HH:mm，如 "08:00"
- `type=once`：支持自然语言相对时间
  - "1分钟后"、"5分钟后"、"30分钟后" → 自动计算绝对时间
  - "1小时后"、"2小时后" → 自动计算绝对时间
  - 也支持 HH:mm 格式，如 "14:51" → 今天的 14:51

### prompt 怎么写
- prompt 是到点后发送给用户的文字，要简短自然
- 例如："提醒用户喝水" → 到点后推送这句话
- 不要写复杂的执行指令，只是消息文本

---

## 第二部分：执行定时任务（Worker 对系统）

当定时任务到点触发时适用。

### 用户创建的提醒（system=false）
- 直接取 prompt 文字发送给用户
- 不做任何 LLM 调用，不执行任何额外操作
- 一次性任务执行后自动删除

### 系统 Worker 任务（system=true，如 daily-worker）
- 先调 `load_skill("daily-simulation")` 加载日常模拟规则
- 再调 `load_skill("scheduled-task")` 加载本 Skill
- 读 `.paicli/souls/` 下的角色文件
- 查 `timeline-life.md` 判断今天有无事件
- 没事件就按 daily-simulation 规则编日常
- 有事件就推进事件
- 写日志到 `logs/` 目录
- 执行完就结束，不要额外创建定时任务
