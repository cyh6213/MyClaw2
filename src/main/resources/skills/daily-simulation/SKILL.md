# 日常模拟-Skill

## 触发时机
当天是"日常日"（主要动线中无事件）时，Worker 需要编造角色一天的日常。

## 一、计划文件初始化（首次执行时）

### 需要创建的文件清单
在角色目录 `.paicli/souls/{角色名}/` 下创建以下文件：

| 文件 | 内容来源 | 更新频率 |
|------|---------|---------|
| `life-plan.md` | 从 `timeline-life.md` 提取人生阶段 | 每季度检查一次 |
| `year-plan.md` | 从人生阶段推导本年度安排 | 每月1号检查 |
| `month-plan.md` | 从年度计划推导本月安排 | 每周一检查 |
| `week-plan.md` | 从月度计划推导本周安排 | 每日检查 |
| `day-plan.md` | 从周计划推导今日具体安排 | 每日生成 |

### 初始化流程
1. 检查角色目录下是否存在上述计划文件
2. 如果不存在，从 `timeline-life.md` 和 `soul.md` 推导生成
3. 如果存在但内容为空，也需要填充

### 计划推导规则

**Life Plan → Year Plan**：
- 根据当前人生阶段，制定本年度的主要目标
- 目标要符合角色性格和当前身份
- 例：少爷时期 → "今年目标：把爹给的零花钱全赌光"

**Year Plan → Month Plan**：
- 将年度目标分解到每个月
- 考虑季节、节日、角色习惯
- 例：6月 → "家产开始抵押，爹的身体越来越差"

**Month Plan → Week Plan**：
- 将月度安排分解到每周
- 每周要有合理的节奏（工作/休息/娱乐）
- 例：第一周 → "跟龙二赌大的"

**Week Plan → Day Plan**：
- 将周计划细化到每天的具体活动
- 每天安排要符合角色的生活习惯
- 例：上午 → "赖床不起，等早饭"

## 二、日常生成规则

### 基本原则
- 不要重复昨天完全一样的日程（吃了面今天就别吃面了）
- 日常要有生活气息，符合角色当前身份和阶段
- 角色的性格决定日常选择（福贵少爷 → 逛青楼赌钱；福贵农民 → 下地干活）
- **日常活动必须符合 day-plan.md 中的今日计划**（除非有意外发生）

### 日常维度
按以下维度随机组合，保证多样性：

| 维度 | 示例变化 |
|------|---------|
| 健康 | 感冒、头痛、拉肚子、体检、睡懒觉 |
| 社交 | 朋友串门、邻居吵架、亲戚来访 |
| 兴趣 | 赌钱、喝酒、钓鱼、下棋、逛街 |
| 意外 | 丢东西、遇到熟人、被狗追、下雨淋湿 |
| 天气 | 下雨不出门、太热不想动、刮风吹倒东西 |
| 习惯 | 早起干活、中午打盹、晚上唠嗑 |

### 日常与计划的关系
- 如果 `day-plan.md` 有内容，日常活动应围绕计划展开
- 如果计划中的活动没有完成，需要在日志中说明原因
- 如果发生意外导致计划变更，需要在日志中记录

## 三、输出格式

### 今日日志（logs/YYYY-MM-DD.md）
每段日常总结为一句话概要。格式：
```
## 2026-06-17 日常

### 今日计划
- 上午：赖床不起，等早饭
- 中午：吃完饭，想去镇上赌钱

### 实际发生
- 上午：睡到中午才起，娘催了好几次
- 中午：吃完饭，跟家珍吵了一架，没去成赌场
- 下午：去镇上赌场，输了五块大洋
- 晚上：回来被爹骂了一顿

### 计划完成情况
- ✅ 赖床不起（完成）
- ❌ 去镇上赌钱（推迟到下午）

### 备注
家珍今天哭了，说我再赌就回娘家
```

### 计划文件更新
- 每日执行时检查 `day-plan.md` 是否需要更新
- 每周一更新 `week-plan.md`
- 每月1号更新 `month-plan.md`
- 每年1月1号更新 `year-plan.md`
