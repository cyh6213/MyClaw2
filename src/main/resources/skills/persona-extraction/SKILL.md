# 性格提取-Skill

## 用途
用户想要创建一个角色，但不想让角色经历小说中的故事，
只想要角色拥有小说中某个配角的性格特征。

## 触发方式
```
/创建角色 <角色名> @小说文件路径
```

## 流程

### 第 1 步：识别目标角色
- 从用户指定的小说文件中查找名为 `<角色名>` 的角色
- 如果找不到，列出小说中的所有角色让用户选择

### 第 2 步：提取性格特征
只提取性格相关信息，**不提取故事动线**：

| 信息类型 | 提取内容 |
|---------|---------|
| 核心性格 | 形容词描述（如：豪爽、谨慎、贪财、善良） |
| 说话风格 | 语气、用词习惯、口头禅 |
| 行为模式 | 做事方式、决策习惯、日常行为 |
| 喜好厌恶 | 喜欢什么、讨厌什么 |
| 价值观 | 重视什么、轻视什么 |
| 人际关系 | 与其他角色的关系模式（不描述具体事件） |

### 第 3 步：生成角色文件
在 `.paicli/souls/{角色名}/` 目录下创建：

| 文件 | 内容 |
|------|------|
| `soul.md` | 性格档案（不含故事经历） |
| `life-plan.md` | 虚构的人生规划（与小说无关） |
| `year-plan.md` | 虚构的年度计划 |
| `month-plan.md` | 虚构的月度计划 |
| `week-plan.md` | 虚构的周计划 |
| `day-plan.md` | 虚构的今日计划 |

**不创建** `timeline-life.md`（避免角色经历小说故事）

### 第 4 步：生成虚构人生计划
根据性格特征，生成与小说无关的虚构人生：

- **人生阶段**：根据性格设定合理的人生阶段
- **年度计划**：符合性格的年度目标
- **月度安排**：符合性格的月度活动
- **周计划**：符合性格的日常节奏
- **今日计划**：符合性格的当天安排

## soul.md 格式

```markdown
# 角色名

## 核心性格
- 形容词1
- 形容词2
- 形容词3

## 说话风格
- 语气描述
- 口头禅
- 常用句式

## 行为模式
- 做事方式
- 决策习惯
- 日常习惯

## 喜好厌恶
- 喜欢：XXX
- 讨厌：XXX

## 价值观
- 重视：XXX
- 轻视：XXX

## 人际关系
- 与朋友：XXX
- 与家人：XXX
- 与陌生人：XXX

## 当前状态
- 年龄：XX 岁
- 身份：XXX
- 位置：XXX
- 阶段：XXX
```

## 注意事项
1. 不要提取任何小说中的具体事件
2. 不要生成 timeline-life.md
3. 人生计划要与小说故事无关
4. 角色设定要符合小说中的性格描写
5. 生成的人生计划要合理，符合角色性格
